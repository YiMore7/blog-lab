# 问题陈述：浏览器重要日期倒计时插件

## 1. 核心问题 (Problem Statement)

* **我们试图解决什么问题？**
  用户经常忘记重要日期（生日、纪念日、项目截止日期、考试日期等），现有的日历提醒方式过于复杂或容易被忽略，需要一个轻量级、随时可见的倒计时提醒工具。

* **这个问题为什么重要？**
  错过重要日期会导致人际关系受损、工作延误、学习机会丧失，造成情感和经济损失。现有解决方案要么太重（需要打开应用），要么太轻（容易被忽略）。

## 2. 目标用户 (User Persona)

* **主要目标用户是谁？**
  - 忙碌的职场人士：需要跟踪项目截止日期、会议、客户生日
  - 学生群体：考试日期、作业截止日期、重要活动
  - 家庭管理者：家庭成员生日、纪念日、重要缴费日期
  - 自由职业者：客户交付日期、发票截止日期

* **他们的核心痛点/诉求是什么？**
  - 需要一个不打扰、但随时可见的提醒
  - 希望重要日期始终在视线范围内
  - 不希望被复杂的日历应用分散注意力
  - 需要快速添加和编辑重要日期

## 3. 核心价值 (Value Proposition)

* **我们提供的核心解决方案是什么？**
  一个浏览器插件，在浏览器工具栏显示最重要的日期倒计时，点击可查看所有重要日期的列表，支持快速添加和自定义提醒。

* **这对用户的核心价值是什么？**
  - **零认知负担**：重要日期始终在视线范围内，无需主动记忆
  - **零打扰提醒**：静默倒计时，不弹出干扰通知
  - **极速操作**：2秒内添加新日期，1秒查看所有日期
  - **个性化体验**：支持自定义显示样式和重要程度分级

## 4. 关键假设 (Key Assumptions)

* **我们做了哪些核心假设？**
  - 假设用户愿意在浏览器中安装插件来管理重要日期
  - 假设用户更喜欢静默提醒而非主动通知
  - 假设用户的重要日期数量在20个以内（太多会失去焦点）
  - 假设用户主要在桌面环境使用浏览器工作/学习

* **哪些假设如果错误，会导致项目失败？**
  - 如果用户更喜欢手机APP而非浏览器插件，使用率会很低
  - 如果用户需要复杂的重复提醒功能（如每周、每月），当前设计无法满足
  - 如果用户对隐私敏感，不愿意将重要日期存储在浏览器中

## 5. 成功指标 (Success Metrics)

* **北极星指标：** 日活跃用户数（DAU）中，每日主动查看倒计时的比例 > 40%
* **一级指标 (输出)：** 
  - 用户添加的第一个日期后7天留存率 > 60%
  - 平均每用户添加的重要日期数量 > 3个
  - 用户平均每日查看倒计时次数 > 2次
* **反向指标 (制衡)：**
  - 插件卸载率 < 5%/月
  - 用户投诉"干扰性"的比例 < 1%

## 6. 范围边界 (Scope & Non-Goals)

* **必须包含 (Must-Have)：**
  - 浏览器工具栏图标显示最近重要日期的倒计时
  - 点击图标展开所有重要日期的列表视图
  - 快速添加新日期（日期、标题、重要性级别）
  - 基础自定义（显示格式、颜色主题）
  - 数据本地存储（不依赖云服务）

* **明确不做 (Non-Goals)：**
  - 不支持重复事件（如每周会议）
  - 不支持团队协作或分享功能
  - 不支持移动端同步
  - 不支持复杂的时间管理功能（如番茄钟、任务清单）
  - 不支持声音或弹窗提醒（保持静默）
