对于数据，我有这样几个理解：

第一，数据不等同于真相，真相往往比数据更加复杂，研究人员需要看到的是具象化的真相，而不是抽象的数据

第二，数据本身没有观点，研究人员不能预设观点、只喜欢那些能够支持自己观点的数据

第三，数据不一定永远有用，不同情况下，一些曾经有用的数据可能不再有用，需要找到新的指标。

关于用户标签，按照计算方式不同，标签可分为4类

- 统计类：最常见的一类标签，不需经过任何二次计算即可得到，如用户地区、年龄段等
- 规则类：基于业务的数据规则而定，如用户分层中的活跃用户、流失用户等
- 模型类：需要通过建立模型进行数据计算，如用RFM模型【最近一次消费 (Recency)、消费频率 (Frequency)、消费金额 (Monetary)】
- 预测类：根据搜集到的数据，对未来可能发生的事件或者行为进行预测，如流失预警、续报概率等

常用的机器学习模型和算法：

1、感知机学习算法。如何在一堆数据中找到一个界限把量化总不同分类的数据划分开，二分法的思路，不断调整划分界限 直到数据都被准确划分。但是这种分类是会错的，它是一种非正即负的思路

2、线性回归模型。通过观测数据来拟合一条直线，通过该直线可以看出未来数据的变化趋势及具体的值，比如天气预报

3、逻辑回归。一种更广义的线性回归，更连续、更平顺的方式预测某个实例的走向，甚至能给出概率值。而相比感知机学习算法，给出走向和概率，直接按照历史数据建模，从工程学上 更便捷迭代快

上述3着都适用于二分类问题。

4、K近邻算法。适用于多分类问题。遍历每个输入点 找到最接近的某个类。像推荐系统中向用户推荐物品的时候，通常先看社交关系链用户对哪些物品感兴趣

5、朴素贝叶斯分类器。条件概率分类器。适用于分类预测、文本分类、垃圾邮件过滤、个性化推荐等场景，不适合相关性高的

我们把“从获取信息到采取行动”这个过程，当做一个认知行为的最小闭环，它特别像一个四人接力赛：

第一环节是感知。 当一件事情发生的时候，你首先要从外界去获取信息，这时你要充满好奇心。

第二环节是认知。 你要把各种可能性都罗列出来，评估每种可能性发生的概率。这时你要能保持灰度，接受各种不同的观点，哪怕是你不喜欢的。

第三环节是决策。 你必须作出黑白分明的选择。即使你没有把握，你也要发出清晰的指令。

第四环节是行动。 你就变成了一个坚定的执行者，就像闯进了瓷器店的大象，要勇往直前地完成任务。

所以，这个接力赛是由这四个人完成的，他们分别叫 “好奇感知”、“灰度认知”、“黑白决策”和“疯子行动”。

假设检验试验/显著性水平

实验结束后，结论只是一个基于试验样本的“假设”，但我们其实更想知道的是总体怎么样。为了提升结论的可靠性，我们会基于对这个“假设”进行“检验”，看看这个“假设”在应用到“总体”时是否靠谱。

统计假设检验也称为显著性检验，即指样本统计量和假设的总体参数之间的显著性差异。假设检验运用了小概率原理，事先确定的作为判断的界限，即允许的小概率的标准，称为显著性水平。如果根据命题的原假设所计算出来的概率小于这个标准，就拒绝原假设，认为有显著性差异；大于这个标准则不拒绝原假设，认为无显著性差异

估计总体参数落在某一区间内，可能犯错误的概率为显著性水平，用α表示。α表示原假设为真时，拒绝原假设的概率。1-α 为置信度或置信水平，其表明了区间估计的可靠性

第三步，消除变量C

非暴力沟通的第一个要素是观察。将观察和评论混为一谈，别人就会倾向于听到批评，并反驳我们。

非暴力沟通是动态的语言，不主张绝对化的结论。它提倡在特定的时间和情境中进行观察，并清楚地描述观察结果。

了解他的观察、感受、需要和请求后，我问自己是否愿意满足他的请求

第二个要素是感受。我们的需要和期待，以及对他人言行的看法，导致了我们的感受

一旦认为自己受到了指责，他很可能就会觉得委屈并退缩，这样，双方的关系甚至会更加疏远。但是批评往往暗含着期待。对他人的批评实际上间接表达了我们尚未满足的需要

“如果我们通过批评来提出主张，人们的反应常常是申辩或反击。反之，如果我们直接说出需要，其他人就较有可能作出积极的回应”

第三个要素是需要。一旦人们开始谈论需要，而不指责对方，他们就有可能找到办法来满足双方的需要

第四个要素是请求。我们告诉人们，为了改善生活，我们希望他们做什么。我们避免使用抽象的语言，而借助具体的描述，来提出请求

请求他人的反馈 来确认理解一致

不要让强人所难。在提出请求前，如果我们带着自己预设的想法，那么，我们的请求已经成为了对他人的要求。如果我们有这样的想法，一旦别人没有满足我们的要求，我们难免就会指责他们

## 2023-06-13 08:31:13

倾听并不是对方说什么你听了就完了

可能对方的话带着情绪 富有攻击性，可能周围环境下的人都沉默了

而是要揣摩对方的感受和需要 再给予反馈

而不是凭自己的直觉 评头论足 要求改变 或替对方担惊受怕

要去体会对方 而不是下结论

在倾听他人的观察、感受、需要和请求之后，我们可以主动表达我们的理解。如果我们已经准确领会了他们的意思，我们的反馈将帮助他们意识到这一点

反馈的最好表达方式是疑问句，但也不能太直接 会有距离感。可以先提及自己的感受和需要，再提问。比如：“你说的是什么事？” 与 “我有些困惑。我想知道你是指哪件事。告诉我好吗？”

在给他人反馈时，我们的语气十分重要。一个人在听别人谈自己的感受和需要时，将会留意其中是否暗含着批评或嘲讽。如果我们的语气很肯定，仿佛是在宣布他们的内心世界，那么，通常不会有好的反应。然而，一旦别人通过我们的语气意识到我们是在体会，而非下结论，他们一般就不会产生反感。

愤怒是我们的思维方式造成的。它的核心是尚未满足的需要。如果我们能够借助它来提醒自己——我们有需要没有得到满足，而我们的思维方式正使它难以得到满足，那愤怒就是有价值的。

因为愤怒驱使我们去惩罚他人，而不是去满足需要。

为此，我建议，与其沉浸于“合理的愤怒”，不如倾听自己和他人的需要。这也许需要一个过程，但通过不断的实践，我们将会有意识地用“我生气是因为我需要…… ”来取代“我生气是因为他们”

表达愤怒的四个步骤是：

（1）停下来，除了呼吸，什么都别做

（2）想一想是什么想法使我们生气了

（3）体会自己的需要

（4）表达感受和尚未满足的需要

有时，在第3步和第4步之间，我们需要先倾听他人。在得到倾听和理解之后，他们也就可以静下心来体会我们的感受和需要

多巴胺的分泌来自大脑对“意外”的反应。比如你看短视频的时候，刷新一下，看到一条搞怪视频，再刷新一下，又看到一个名人分享……这些内容就像盲盒，不断刺激大脑分泌多巴胺。由于多巴胺可以让人感到快乐，于是我们会不由自主、不加控制地获取更多信息……而我们的大脑，也容易变成一台“不分青红皂白的好奇心机器”。

了解多巴胺在奖赏回路中的作用可以帮助我们更好地管理自己的行为，尤其是在面对可能导致沉迷的情况时。有一些方法可以帮助减少过度依赖多巴胺刺激而导致的沉溺行为：

- 设定时间限制：
为社交媒体、视频等消费设置明确的时间限制，这可以帮助你更好地控制自己的使用习惯。例如，设定每天只花一定时间在这些平台上，避免过度沉迷。

- 规划活动：
制定有意义的日常活动和计划，比如运动、阅读、学习新技能等，让自己的生活更多元化。这样可以减少对盲盒式刺激的依赖，让大脑得到其他类型的满足感。

- 注意思维模式：
意识到多巴胺的影响并尝试改变思维模式，意识到短期愉悦并非长期满足。思考行为的长期影响，这有助于避免只为了短暂的愉悦而沉迷于某种行为。

- 断舍离和限制访问：
删除不必要或过于频繁引发多巴胺释放的应用，减少这些刺激的来源。同时，设置应用限制或使用应用程序来限制访问时间，帮助自己控制使用频率。

- 寻求支持：
与朋友、家人或专业人士讨论，寻求支持和建议。有人陪伴或理解，有时能够帮助我们更好地控制自己的行为。

布鲁姆的认知领域分类体系，将认知过程分为六个层次，从低到高分别是：记忆、理解、应用、分析、综合、评价

记忆（Knowledge） - 回忆、识别和记住已学习的信息和事实

理解（Comprehension） - 解释、概括信息，以及对所学内容进行解释

应用（Application） - 将学到的知识应用到新的情境中

分析（Analysis） - 将复杂的信息分解成部分，理解这些部分之间的关系

综合（Synthesis） - 将不同的信息和概念组合成一个整体，创造出新的模式或结构

评价（Evaluation） - 做出判断，评估信息、方法或材料的价值

皮亚杰将儿童的认知发展分为以下六个阶段：

- 感觉运动阶段（Sensorimotor Stage）
- 从出生到大约2岁。在这个阶段，儿童通过感官和运动能力来了解世界，学习物体永久性的概念。

- 前运算阶段（Preoperational Stage）
- 从大约2岁到7岁。在这个阶段，儿童开始使用符号（如言语和图像）思考，但他们的思维还是以自我为中心的，缺乏逻辑运算能力。

- 具体运算阶段（Concrete Operational Stage）
- 从大约7岁到11岁。这个阶段的儿童开始发展逻辑思维能力，能够进行具体物体的分类、排序和量化。

- 形式运算阶段（Formal Operational Stage）
- 从大约11岁到成年。在这个阶段，青少年开始能够进行抽象思维，逻辑推理，以及系统地解决问题。

- 后形式思维阶段（Postformal Operational Stage）
- 通常在青少年晚期或成年早期开始。这个阶段的个体能够超越具体的事实和规则，进行更复杂的思考，如反思、批判性思维和问题解决。

- 认知复杂性阶段（Cognitive Complexity Stage）
- 这是后期对皮亚杰理论的扩展，认为成年后个体的认知能力会继续发展，包括更高级的思维能力，如抽象概念的整合、复杂问题的解决等重试重试

## 2024-05-24 09:32:27

大模型的上下文窗口单位多少多少K，代表什么？

1K = 1000字节（bytes），如果一个AI模型支持4K的上下文窗口，那么它能够一次性处理的文本量大约是：4K×1000 bytes=4000 bytes

在文本处理中，一个英文字符通常占用1个字节，而中文字符可能占用2-3个字节，具体取决于编码格式（如UTF-8）。因此，4K的上下文窗口大约可以处理4000个英文字符或更少的中文字符

这个是越大越好吗？不是

越来越大的K带来的优势：
- 理解上下文
：AI模型在处理文本时，需要理解文本中的语义和上下文关系。上下文窗口越大，模型能够一次性处理的文本信息就越多，有助于更好地理解文本。
- 记忆能力
：上下文窗口也反映了模型的短时记忆能力。窗口越大，模型能够记住的信息就越多，有助于捕捉文本中的长距离依赖关系。
- 处理长文本
：对于长文本，如果上下文窗口较小，模型需要分多次处理，可能导致信息丢失和理解不准确。较大的上下文窗口有助于更好地处理长文本。

弊端：

较大的上下文窗口可以提高模型的性能和效率，但同时也会增加计算资源的需求。需要在窗口大小和计算资源之间找到平衡。

形容大模型多少B的是什么

在人工智能和机器学习领域，一个“B”代表10的9次方，即1亿个参数。因此，当我们说一个大模型有多少B时，我们实际上是在询问该模型具有多少亿个参数

一个模型的参数规模可以用“B”（Billion，即亿）来表示，这里的“B”代表了模型中权重和偏置的总数。在Llama 3模型中，8B和70B分别表示模型有80亿和700亿个参数

参数越多，模型通常能够表示更复杂的函数，理论上能够捕捉更细微的数据特征和模式。然而，这也意味着模型需要更多的数据来训练，以避免过拟合，并且需要更多的计算资源

我们当前AI所处的阶段：

AI的点解决方案是用AI改善你现有的决定，应用解决方案是用AI改变你做决定的方式，系统解决方案是 AI促成 了新的决定———你的生产模式整个变了

- 点解决方案（The Point Solution）

- 定义
：点解决方案是指简单的输入端替换，即用AI来改善现有的决定或流程。

- 例子
：

- 亚马逊的商品推荐
：亚马逊的AI根据用户的购物喜好推荐商品，这是一种点解决方案，因为它只是改进了用户的购物体验，而没有改变用户的购物方式。

- 保险理赔自动化
：AI通过分析用户上传的照片来快速估价并处理理赔，这同样是点解决方案，因为它只是提高了理赔效率，而没有改变保险公司的整体业务流程。

- 应用解决方案（The Application Solution）

- 定义
：应用解决方案是指用AI来改变做决定的方式，即对现有的生产装置或业务流程进行改造。

- 例子
：

- 温室虫害预测
：利用AI提前一周准确预测温室是否会长虫，农民可以提前订购抗虫用品。这比单纯的点解决方案更进一步，因为它涉及到对现有农业流程的部分改造。

- 个性化学习
：AI根据学生的学习情况调整教学内容，确保学生在“学习区”内学习，从而提高学习效率。这虽然改进了教学方式，但学校的组织形式（按年龄分级）并未改变。

- 未达到的阶段：系统解决方案（The System Solution）

- 定义
：系统解决方案是指AI促成新的决定，导致整个生产模式和组织方式的系统性变革。

- 例子
：

- 电力革命
：电力的广泛应用不仅改变了动力来源，还改变了整个生产方式和组织方式（如生产流水线的出现）。

- 农业革命
：如果AI不仅能预测虫害，还能让农民不再担心虫害，从而种植更多种类的农作物并扩大温室规模，这就是系统解决方案，因为它彻底改变了农业的生产方式。

这个和常规的理解是有点相悖的，但其实没有理解背后的“设计理念”：

法律中的模糊性并非出于各方利益的妥协，而是为了在实际应用中具有更大的灵活性和适应性。以下是对这句话的详细解释：

1. 灵活性和适应性

- 应对变化
：法律需要应对社会、经济和技术等方面的变化。如果法律规定得过于具体和僵化，就难以适应新的情况和挑战。

- 地区差异
：不同地区在经济发展水平、文化背景和社会习惯等方面存在差异。模糊的规定可以给地方或具体情况更多的调整空间，使其更具操作性。

2. 实用性

- 操作便利
：模糊的规定可以让法律在实际执行中更具灵活性，避免因过于具体而导致的执行困难。

- 案例裁决
：模糊的法律条文可以为法官和执法者提供一定的裁量权，使他们能够根据具体情况做出公正的裁决。

3. 避免僵化

- 防止过时
：法律如果过于具体，可能会很快过时，无法应对新的社会现象和问题。模糊的规定可以使法律具有更长的生命力。

- 促进创新
：模糊的法律条文可以为新技术和新模式的发展提供空间，避免因法律规定过于严格而抑制创新。

如何衡量AI生成的质量？

- 人工标注，作为验证集，衡量AI生成的质量

- 合格的质量标准

- 准确率：整体预测对的比例，回答“总共对了多少”。

- 精准率：：预测为正类的样本中，真的正类有多少，回答“正类预测靠不靠谱”。

- 召回率：所有真实正类中，模型找出了多少，回答“正类有没有漏掉很多”。

- 精准率

指被预测正确的正样本占全部被预测为正的样本的比重，它衡量模型返回的答案中有多少是正确的，高精确率意味着模型给出的答案中，大多数是正确的。其计算公式为：Precision = 正确的回答数 / 返回的总回答数

在当前的 RQI 研发中，合格的质量标准要求精准率 precision >= 80% ，即模型预测为正的样本中，至少有 80% 是正确的，才符合上线到生产环境的条件

- 召回率

一个衡量模型性能的指标，指在实际为正的样本中被预测为正样本的概率。在不同场景下，召回率的计算方式和应用有所不同

在模型预测中
：某类别的召回率指该类别预测正确的个数占某类别的总数的比例。例如，类别为 0 的数据总共 5 条，其中 3 条预测正确，召回率为 60%；类别为 1 的数据总共 5 条，其中 4 条预测正确，召回率为 80%

“结构性制度障碍”之所以是消费困境的根源，是因为它们直接决定了资源分配、社会保障和个人消费能力的天花板。

让我们拆解一下原因：

1. 户籍制度：身份决定机会

中国的户籍制度将人口分为“城市”和“农村”两大类，决定了教育、医疗、住房等公共资源的分配。比如，农民工即使在城市工作多年，子女却难以享受同等的教育和医疗资源。这种“身份壁垒”让一部分人始终处于消费能力的下层，消费意愿和能力都被压制。

2. 社会保障体系：安全感缺失

中国的社保体系分层明显，城市职工和城乡居民的养老金、医保待遇差距极大。数据显示，城市职工平均每月可领取3500元养老金，而普通居民仅200元左右，欠发达地区甚至更低。这种不平等让大多数人缺乏“敢花钱”的安全感，储蓄成为防御性选择，消费自然受限。

3. 土地制度：财富流动受限

城市土地归国家所有，农村土地归集体，农民的土地不能自由买卖或抵押，导致农村居民难以通过土地变现改善生活。土地不能流通，意味着财富无法有效转化为消费能力，农村人口的消费潜力被长期锁死。

4. 三重障碍叠加，形成“隐形贫困”

这些制度障碍像“看不见的围墙”，把社会分割成不同的消费阶层。即使经济总量增长，底层群体因身份、保障、资产流动性受限，始终难以享受消费升级的红利。这种“制度性贫困”不是一时的收入问题，而是结构性、代际性的困境。

5. 只有制度松绑，消费才能真正释放

作者强调，所有短期刺激政策都只是“头痛医头”，只有打破户籍、社保、土地等结构性障碍，才能让更多人有信心、有能力消费，经济才有真正的内生动力。

类比说明：

可以把中国的消费困境想象成一条大河，河流上有三道闸门（户籍、社保、土地），只要闸门不打开，哪怕上游水量再大，下游也只能涓涓细流。只有把闸门打开，水流（消费力）才能奔涌而下，滋养整个经济生态。

总结：

结构性制度障碍不是表面的小石头，而是深埋河床的“硬土层”，只有动大手术，才能让消费真正成为经济的“救命稻草”。

Agent 有两个变量，一个是控制任务走向的 workflow 工作流，一个是控制内容生成的 context 上下文。

1）如果 workflow 和 context 的确定性都很高，这类任务就容易被自动化，类似传统 RPA，比如在处理发票处理、表单填报任务时，AI 更多是粘合剂，发挥空间比较有限。

2）如果 workflow 确定但 context 不确定，也就是流程固定但输入多变，就需要 Agent 在语义和理解上补全，比如客服问答、合同解析，需要通过外部检索、知识图谱等工具来弥补信息的缺口，让推理结果更符合预期。

3）如果 workflow 不确定但 context 确定，也就是输入清晰但走法多样，Agent 就要去自主规划路径，例如市场分析报告生成、个性化推荐等，大多数 End-to-End RL Agent 都擅长做这类任务，因为它们在训练阶段就习得了大量的路径规划和解题思路。

4）而当 workflow 和 context 都不确定时，就是最复杂的场景了，既要推理也要探索，像创新方案设计、跨部门信息收集等，这类更偏向于通用型 Agent，它的执行效果，取决于给它配备的工具丰富度，尤其是编程能力要最大化开放，例如让它学会去 Github 找仓库克隆并修改代码来解决问题，让它像人一样干活儿。

所以，要把 Agent 做好，首先要明确场景。本质上，自动化解决的是“确定性”问题，而智能化解决的是“不确定性”问题。

## 2025-10-31 19:26:18

它强调AI产品成败取决于“如何衡量结果”，而非“如何做功能”。

这句话把AI产品经理的核心职责从“堆功能、调交互、看单点指标”转到“构建可被验证的效果标准与测评体系”，用真实的用户感受来驱动pipeline优化与模型精调。我的理解分三层：

本质

AI的输出是概率性的、上下文敏感的，单次demo很容易“看起来不错”。因此，唯一可靠的改进路径是：先定义“好结果”的可操作标准，再用系统化测评去逼近它。标准错了，优化方向就错；测评不客观，就制造“虚假的乐观”，团队在错误的山头上越爬越高。

实践

- 标准：用任务导向的、可复现的结果定义，而非抽象体验标签。比如“会议纪要”要约束准确性、覆盖度、结构化、可执行性，而不是“看起来像总结”。

- 测评：离线基准集+在线A/B结合，覆盖“平均表现”和“长尾坏例”。面向用户的综合分比单一模型分更重要。

- 优化闭环：以测评信号定位问题归因（数据、PE、策略、模型），按投入产出比安排SFT/RL或产品策略，而不是盲目“再训一点”。

含义

- AI产品的护城河来自“效果定义与验证体系”，而不是单一模型或炫技交互。

- PM是“结果的负责人”。你决定“什么叫好”，也就决定团队“往哪儿好”；因此标准与测评是权力的源头，也是风险的源头。

- 这是一种“反功能主义”的方法论：先结果，后实现；先真测，后优化；先用户感受，后模型分数。

一句话复述：在AI里，度量即战略。没有对“结果”的正确度量，就没有正确的产品。
