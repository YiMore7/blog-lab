# 我的AI年度盘点

_2025年12月16日 11:39_ _北京_



  

又到年底盘点的时候了。我准备用两期视频，分别从行业和个人层面做个回顾。本期视频，我先聊聊我个人在过去一年使用AI的收获。

  

做科技博主的一个好处是，有任何的想法、实践，我都会用视频的形式记录下来。我把2024年发的视频和2025年的做了个对比，发现两个非常明显的变化。

  

第一，系统层面。放弃以外挂式RAG的旧体系，转向以Coding Assistant为核心的新体系。

  

熟悉我的小伙伴都知道，我最早做了大量RAG相关的视频。因为我坚信：

  

要让AI有用起来，必须给它补上两个东西——领域知识和领域经验。

  

领域经验落到AI层面，之前是Workflow，现在是Skills。Anthropic推出Skills之后，我做了好多期视频推荐，就是这个原因。

  

领域知识落到AI层面，就是RAG。通过RAG技术，让AI在生成的时候不只是依靠训练时灌进去的通用知识，还可以依靠用户提供的领域知识。

  

2024年，围绕RAG，我研究了好多工具，也发现RAG很大的局限性——这一点，我之前还特意出了视频。

  

转机发生在今年。Vibe Coding火了之后，我开始做产品。当我接触Cursor之后突然发现，它的检索能力和编辑能力，除了用来编程，也完全适合日常工作的场景！

  

你想啊，现在的模型上下文更长了，像Cursor这样的原生工具已经把检索内化得足够好了，不需要用户再去操心向量数据库、切片这些中间环节了。

  

于是我把Obsidian和Cursor连到一起。之前我用过很多AI插件，但感觉都差点意思。现在有Cursor了，我发现可以用Cursor打开笔记仓库。这样一来，Cursor就在前端负责编辑，Obsidian退到后端负责存储。

  

这个思路在后来得到了极大强化。

  

MCP出来之后，让我们可以连接更多工具。比如连接Tavily或者Perplexity的MCP，增强系统的外部搜索能力。

  

Claude Code作为最强Coding Assistant，工具调用强，还可以更换作为驱动引擎的模型，让整套系统有了质的飞跃。

  

而后来Skills的推出，让我们可以把领域经验封装进去，给到AI。这样一来，我之前说的领域知识和领域经验都齐全了！

  

事实证明，我的这套思路是非常正确的。国内外有不少小伙伴也萌生了同样的想法。Anthropic把Claude Code SDK改名Claude Agent SDK。官方在博客文章里说了，这套东西可以用在更广的范围，不只是编程。

  

我前段时间出的关于AI OS的视频，就是这个思路的阶段性总结。如果你像我一样非常在乎生产效率的最大化，在乎个人AI系统的搭建，请一定试一试我的这个方法。

  

第二，工具层面。全面拥抱Google生态。

  

我是从Gemini 2.0时期开始把这个模型作为日常主力的。因为它真的太全面了。越用越喜欢。而且有了模型加持之后，Google生态内的那些产品都有了质的飞跃。

  

比如新出的NotebookLM，是目前公认最强的AI学习工具。那些老的产品，比如Gmail、Docs，都接入Gemini、并且横向打通之后，生产效率和体验提升了非常多。

  

之前我介绍过的AI OS，依托于Claude Code框架，只适合在桌面端使用。Google生态的这些产品却覆盖从桌面端到移动端，很适合拿来满足我轻量化的需求。

  

目前的AI御三家——Google、Anthropic和OpenAI，在C端，我还是坚定看好Google。我不止一次在强调，原生多模态路线跑通之后，会给模型和产品带来极大提升。

  

所以我在星球内说过，接下来我会逐渐把自己积攒的资料往Google Drive里存一份。不管是Gemini还是NotebookLM，都可以从Google Drive导入资料。或者不导入也行，在Google Docs里打开Gemini就可以结合文档内容进行对话。

  

而在B端，我看好Anthropic。就像前边说的，Claude Code这套东西已经是通用Agent框架了。这个Anthropic自己吹的，而是他们看到大量用户就是在这么用的。对于企业来说，要开发Agent，与其自己搭建框架，还不如用他们的SDK更划算。

  

至于OpenAI，我觉得他们现在是多线作战，有点消耗不起了。

  

OpenAI现在在跟Google比多模态、比生态，跟Anthropic比编程能力、比Agent能力，还要搞大基建。四面开战，想撑住估值。但是从最近几个月的情况来看，战略失焦导致OpenAI已经有点吃力了。

  

作为用户，即使抛开对产品的喜好，我也不敢选择OpenAI。就像我在星球里说的，选定一个平台，把数据资产放上去，并且长期使用让模型越来越了解你，这个是不能随便迁移的。

  

所以在工具层面，我应该会一直留在Google阵营。

  

以上这两个层面的东西，就是我在2025年的最大收获。如果你一直在看我的视频的话，就知道我今年的内容都是围绕这两个层面展开的。

  

相比2024年，我没那么频繁使用开源模型，也没那么热衷于部署各种开源项目。这说明，我的整套思路开始成熟了、收敛了。更多精力收回来去打磨自己的系统，而不是到处玩新的工具。
