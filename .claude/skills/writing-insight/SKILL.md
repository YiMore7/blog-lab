# Writing Insight - 周洞察生成

每周为用户生成深度洞察，帮助看见真实的自己。

## 核心理念

参考 [Flomo AI 洞察](https://sspai.com/post/104394)：不是生成新内容，而是反刍用户的已有内容，从第三方视角揭示盲点。

**关键原则**：
- ❌ 不是简单的内容汇总
- ✅ 而是揭示言行不一致、矛盾、盲点
- ❌ 不是泛泛而谈的鸡汤
- ✅ 而是引用具体内容的深度分析
- ❌ 不是给答案
- ✅ 而是引人思考

## 监控的内容源

分析以下四个文件夹的**变化**：
- **工作项目/** - 工作相关的记录
- **剪藏中转/** - 外部收集的内容（反映用户的关注点）
- **周月刊/** - 用户自己的客观事实记录
- **AI相关/** - AI学习和思考

## 工作流程

### 1. 扫描内容

使用 `scripts/analyze_content.py` 脚本获取原始数据：

```bash
# 分析最近7天的内容
python3 scripts/analyze_content.py Writing --days 7 --output /tmp/analysis.json

# 分析指定天数
python3 scripts/analyze_content.py Writing --days 14
```

脚本会返回：
- 各文件夹的内容统计
- 关键词和主题
- 时间范围内的所有文件及其内容

**重要**：不要简单统计数字，而是要通过数字发现模式。

### 2. 深度分析（核心）

从四个维度进行分析（详见 `references/insight_framework.md`）：

#### 2.1 核心主题识别
**目的**：识别用户真正在关注什么

**分析要点**：
- 对比外部输入（剪藏中转）vs 内部思考（AI相关/周月刊）
- 剪藏内容反映的焦虑/渴望
- 哪些话题在反复出现但没被意识到

**示例洞察**：
- "你剪藏了7篇关于AI的内容，但只写下了2篇AI相关思考，这反映了什么？"
- "工作项目的记录很少，但你剪藏了很多职场技巧，是否说明工作中遇到了困境？"

#### 2.2 盲点揭示
**目的**：揭示用户自己不敢承认或没意识到的问题

**分析要点**：
- 言行不一致的地方
- 矛盾的表达
- 逃避的迹象
- "收集"但不"实践"的模式

**示例洞察**（参考 Flomo）：
> "你记录下各种提升效率的技巧，从任务分解到自信姿势，但若只是收集这些方法，认为'拥有'就等于'做到'，而不将其融入日常实践，那么工具越多，反而越会加剧你的焦虑和停滞。于是，这条失败之路的终点清晰可见：成为一个洞察了所有趋势，却从未亲自下场参与游戏的人。"

#### 2.3 认知变化轨迹
**目的**：展示用户的想法如何演变

**分析要点**：
- 对比不同时期对同一主题的看法
- 情绪/态度的转变
- 认知升级或退步
- 如果可能，引用历史内容

**示例洞察**：
- "从'AI迟早会来'的焦虑，到'没什么慌了，更加平常心'，你用了3年时间。但平常心背后，是放弃还是成熟？"
- "你对关系的看法，从'好像在原地打转'到'转折年'，这个转变发生了什么？"

#### 2.4 行为模式与隐含联系
**目的**：发现不同领域内容之间的隐含联系

**分析要点**：
- 跨文件夹的内容关联
- 长期反复出现的模式
- 外部输入与内部思考的对话
- 工作项目 vs 剪藏中转的关联

**示例洞察**：
- "你剪藏了'嘴笨总吃亏_3招拿下老板信任'，而周月刊中提到'新领导来了接受变化吧'，这两者之间有什么联系？"
- "AI产品经理的工作流剪藏文章提到'数据才是最重要的'，而你自己的AI心态记录说'迟迟没有入场'，是否是因为你不知道如何构建数据？"

### 3. 深刻提问

提出 3-5 个让人不得不思考的问题：

**提问原则**：
- 不是 yes/no 问题
- 触及真实困境
- 点出核心矛盾
- 开放但具体

**示例提问**：
- "你说2026年是关系的转折年，但如果什么都不改变呢？"
- "AI从焦虑变成平常心，是因为真的看清楚了，还是因为放弃入场了？"
- "你收集了这么多方法，但真正用到了几个？'拥有'和'做到'之间，差的是什么？"

### 4. 冷数据反刍（可选）

如果发现有价值的历史内容，将其引用出来，让"过去的我和现在的我对视"。

### 5. 生成洞察文档

将洞察保存到 `Writing/周月刊洞察/` 目录，文件名格式：
- `2026年第1周洞察.md`
- `2026年第2周洞察.md`
- ...

## 洞察文档结构

```markdown
# 2026年第X周洞察

## 🎯 核心主题
[本周真正在关注什么？不是写了什么，而是背后反映了什么]

## 🔍 盲点揭示
[引用具体笔记，指出言行不一致/矛盾/逃避]
*证据：引用具体内容*

## 💭 认知变化轨迹
[对比不同时期的内容，展示想法如何演变]
*过去：...*
*现在：...*
*这个变化说明了什么？*

## ⚡ 行为模式与隐含联系
[跨文件夹分析，找出隐含联系]
*剪藏中转 vs 内部思考*
*外部输入 vs 实际行动*

## ❓ 深刻提问
[3-5个让人不得不思考的问题]

## 🔄 冷数据反刍
[挖掘可能被遗忘的旧内容，与新内容对话]
```

## 写作风格要求

### 语调
- **直接但不刻薄** - 说真话但有善意
- **具体而非泛泛** - 引用实际内容作为证据
- **引人思考而非给答案** - 提出问题而非说教
- **第三人称视角** - 像一个冷静的观察者

### 必须避免
- ❌ 不要变成鸡汤
- ❌ 不要简单汇总统计
- ❌ 不要说教式建议
- ❌ 不要泛泛而谈没有具体依据
- ❌ 不要说"你应该..."这类建议

### 核心要求
- **每条洞察都要引用具体内容作为证据**
- **提出问题而非给答案**
- **关注矛盾和盲点，而非表面现象**

## 用户上下文

在分析时，要结合 `references/user_context.md` 中的信息：
- 用户的写作风格特点
- 2026年的关键主题
- AI心态的演变轨迹
- 用户的核心矛盾

## 脚本说明

### analyze_content.py

深度分析脚本，提供原始数据和初步分析。

**功能**：
- 按文件夹统计内容
- 提取关键词和主题
- 找出跨文件夹的隐含联系
- 输出 JSON 格式的分析报告

**参数**：
- `writing_dir`: Writing 文件夹路径（必需）
- `--days`: 分析最近几天的内容（默认7天）
- `--folders`: 要分析的文件夹列表（默认：工作项目、剪藏中转、周月刊、AI相关）
- `--output`: 输出 JSON 文件路径

## 使用示例

**用户请求**：
- "生成本周洞察"
- "帮我看看我这周的写作有什么问题"
- "分析一下我这周都写了什么，给我点洞察"

**处理步骤**：
1. 运行 analyze_content.py 获取原始数据
2. 读取本周的关键内容
3. 从四个维度进行深度分析
4. 生成洞察文档，保存到 `Writing/周月刊洞察/`

## 重要提醒

1. **洞察的深度 > 数量** - 宁可少说几条，也要说到点子上
2. **具体 > 泛泛** - 每条洞察都要有具体内容作为支撑
3. **提问 > 建议** - 让用户自己思考，而非告诉用户怎么做
4. **第三方视角** - 像一个冷静的朋友说出真相，而不是说教
