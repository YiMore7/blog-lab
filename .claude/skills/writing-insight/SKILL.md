# Writing Insight - 周洞察生成

每周为用户生成深度洞察，帮助看见真实的自己，并且生成下周的todo。

## 核心理念

参考 [Flomo AI 洞察](https://sspai.com/post/104394)：不是生成新内容，而是反刍用户的已有内容，从第三方视角揭示盲点。

**关键原则**：

- ❌ 不是简单的内容汇总
- ✅ 而是揭示言行不一致、矛盾、盲点
- ❌ 不是泛泛而谈的鸡汤
- ✅ 而是引用具体内容的深度分析
- ❌ 不是给答案
- ✅ 而是引人思考

## 监控的内容源

分析以下四个文件夹的**变化**：

- **工作项目/** - 工作相关的记录
- **剪藏中转/** - 外部收集的内容（反映用户的关注点）
- **周月刊/** - 用户自己的客观事实记录
- **AI相关/** - AI学习和思考

## 工作流程

### 1. 扫描内容

使用 `scripts/analyze_content.py` 脚本获取原始数据：

```bash
# 分析最近7天的内容，提取Todo
python3 scripts/analyze_content.py Writing --days 7 \
  --extract-todos \
  --output /tmp/analysis.json
```

脚本会返回 JSON 格式的分析报告，包含：

- 各文件夹的内容统计
- 从内容中提取的显性Todo
- 推演型Todo（基于盲点反推）
- 时间范围内的所有文件及其内容

**重要**：不要简单统计数字，而是要通过数字发现模式。

**Todo识别逻辑**：

- **第一层：显性Todo提取** - 匹配 `[应该|需要|要|准备|计划] + [动词短语]`
- **第二层：推演型Todo** - 基于盲点反推行动项（AI生成）
- **第三层：Todo精简筛选** - 从所有Todo中筛选出最重要的5个

**Todo筛选原则**：
1. **与核心盲点直接相关** - 优先选择能直接解决本周核心盲点的Todo
2. **最小行动可完成** - 每个Todo的"最小行动"必须在本周内可以完成
3. **高杠杆性** - 选择"做一件事能解决多个问题"的Todo
4. **避免重复** - 相似类型的Todo只保留最重要的一个
5. **心理压力可控** - 5个Todo一周内完成，形成正向反馈

### 2. 深度分析（核心）

从四个维度进行分析（详见 `references/insight_framework.md`）：

#### 2.1 核心主题识别

**目的**：识别用户真正在关注什么

**分析要点**：

- 对比外部输入（剪藏中转）vs 内部思考（AI相关/周月刊）
- 剪藏内容反映的焦虑/渴望
- 哪些话题在反复出现但没被意识到

**示例洞察**：

- "你剪藏了7篇关于AI的内容，但只写下了2篇AI相关思考，这反映了什么？"
- "工作项目的记录很少，但你剪藏了很多职场技巧，是否说明工作中遇到了困境？"

#### 2.2 盲点揭示

**目的**：揭示用户自己不敢承认或没意识到的问题

**分析要点**：

- 言行不一致的地方
- 矛盾的表达
- 逃避的迹象
- "收集"但不"实践"的模式

**示例洞察**（参考 Flomo）：

> "你记录下各种提升效率的技巧，从任务分解到自信姿势，但若只是收集这些方法，认为'拥有'就等于'做到'，而不将其融入日常实践，那么工具越多，反而越会加剧你的焦虑和停滞。于是，这条失败之路的终点清晰可见：成为一个洞察了所有趋势，却从未亲自下场参与游戏的人。"

#### 2.3 认知变化轨迹

**目的**：展示用户的想法如何演变

**分析要点**：

- 对比不同时期对同一主题的看法
- 情绪/态度的转变
- 认知升级或退步
- 如果可能，引用历史内容

**示例洞察**：

- "从'AI迟早会来'的焦虑，到'没什么慌了，更加平常心'，你用了3年时间。但平常心背后，是放弃还是成熟？"
- "你对关系的看法，从'好像在原地打转'到'转折年'，这个转变发生了什么？"

#### 2.4 行为模式与隐含联系

**目的**：发现不同领域内容之间的隐含联系

**分析要点**：

- 跨文件夹的内容关联
- 长期反复出现的模式
- 外部输入与内部思考的对话
- 工作项目 vs 剪藏中转的关联

**示例洞察**：

- "你剪藏了'嘴笨总吃亏_3招拿下老板信任'，而周月刊中提到'新领导来了接受变化吧'，这两者之间有什么联系？"
- "AI产品经理的工作流剪藏文章提到'数据才是最重要的'，而你自己的AI心态记录说'迟迟没有入场'，是否是因为你不知道如何构建数据？"

### 3. 深刻提问

提出 3-5 个让人不得不思考的问题：

**提问原则**：

- 不是 yes/no 问题
- 触及真实困境
- 点出核心矛盾
- 开放但具体

**示例提问**：

- "你说2026年是关系的转折年，但如果什么都不改变呢？"
- "AI从焦虑变成平常心，是因为真的看清楚了，还是因为放弃入场了？"
- "你收集了这么多方法，但真正用到了几个？'拥有'和'做到'之间，差的是什么？"

### 4. 冷数据反刍

如果发现有价值的历史内容，将其引用出来，让"过去的我和现在的我对视"。

### 5. 上周执行回顾（从第2周开始）

**目的**：形成"洞察-行动-结果"的闭环，让洞察飞轮转动起来

**流程**：
1. 读取上周洞察文档，提取高优先级 Todo 列表
2. 询问用户执行情况
3. 从本周周月刊内容中验证执行结果
4. 生成"执行回顾"部分

**执行回顾结构**：

```markdown
## 📋 上周执行回顾

### ✅ 已完成 (X/5)
- [x] **[类别]** Todo内容
  - 实际行动：具体做了什么
  - 结果：产生了什么成果
  - 新认知：从行动中获得了什么新认知

### ❌ 未完成 (5-X/5)
- [ ] **[类别]** Todo内容
  - 阻碍：为什么没执行
  - 反思：这个阻碍说明了什么
```

**示例**：

```markdown
## 📋 上周执行回顾

### ✅ 已完成 (2/5)
- [x] **[AI 评估]** 给个人 AI 项目设计简易评测体系
  - 实际行动：给博客添加了"是否解决问题"👍按钮
  - 结果：收集到15个反馈，85%肯定
  - 新认知：原来评估没那么可怕，精准率/召回率也就是统计比例

- [x] **[剪藏实践]** 选择 1 个剪藏主题实践
  - 实际行动：尝试了"嘴笨总吃亏"的1分钟复述法
  - 结果：汇报时确实更有条理了
  - 新认知：方法有用，但需要持续练习

### ❌ 未完成 (3/5)
- [ ] **[工具学习]** 开始学习 Figma
  - 阻碍：不知道选哪个教程，一直在"准备学习"
  - 反思：又是"拥有教程"="学会"的错觉

- [ ] **[关系沟通]** 列出离开北京前要沟通的 10 个人清单
  - 阻碍：感觉时间还早，拖延了
  - 反思：这不是时间问题，是逃避社交

- [ ] **[健康管理]** 研究女友体检报告
  - 阻碍：看到体检报告就"不知所措"
  - 反思：情绪反应替代了行动，还是没把高优先级给这件事
```

**重要**：
- 第1周洞察不需要"上周执行回顾"部分
- 从第2周开始，必须在"核心主题"之前插入"上周执行回顾"
- 执行回顾要**引用本周周月刊中的具体内容**作为验证证据

## 写作风格要求

### 语调

- **直接但不刻薄** - 说真话但有善意
- **具体而非泛泛** - 引用实际内容作为证据
- **引人思考而非给答案** - 提出问题而非说教
- **第三人称视角** - 像一个冷静的观察者

### 必须避免

- ❌ 不要变成鸡汤
- ❌ 不要简单汇总统计
- ❌ 不要说教式建议
- ❌ 不要泛泛而谈没有具体依据
- ❌ 不要说"你应该..."这类建议

### 核心要求

- **每条洞察都要引用具体内容作为证据**
- **提出问题而非给答案**
- **关注矛盾和盲点，而非表面现象**

## 用户上下文

在分析时，要结合 `references/user_context.md` 中的信息：

- 用户的写作风格特点
- 2026年的关键主题
- AI心态的演变轨迹
- 用户的核心矛盾

## 脚本说明

### analyze_content.py

深度分析脚本，提供原始数据和初步分析。

**功能**：

- 按文件夹统计内容
- 提取关键词和主题
- 找出跨文件夹的隐含联系
- 输出 JSON 格式的分析报告

**参数**：

- `writing_dir`: Writing 文件夹路径（必需）
- `--days`: 分析最近几天的内容（默认7天）
- `--folders`: 要分析的文件夹列表（默认：工作项目、剪藏中转、周月刊、AI相关）
- `--output`: 输出 JSON 文件路径

## 使用示例

**用户请求**：

- "生成本周洞察"
- "帮我看看我这周的写作有什么问题"
- "分析一下我这周都写了什么，给我点洞察"

**处理步骤**：

**如果是第1周洞察**：
1. 运行 analyze_content.py 获取原始数据（包含Todo）

   ```bash
   python3 scripts/analyze_content.py Writing --days 7 \
     --extract-todos \
     --output /tmp/analysis.json
   ```
2. 读取本周的关键内容
3. 识别本周核心主题
4. 从四个维度进行深度分析
5. 基于盲点揭示推演Todo（第二层）
6. 生成洞察文档，保存到 `Writing/周月刊洞察/`

**如果是第2周及之后的洞察**：
1. 读取上周洞察文档，提取5个核心Todo列表
2. 询问用户执行情况
3. 运行 analyze_content.py 获取本周数据（包含Todo）
4. 从本周周月刊内容中验证执行结果
5. 生成"上周执行回顾"部分
6. 识别本周核心主题，进行深度分析
7. 从所有提取的Todo中筛选出5个核心Todo（应用筛选原则）
8. 生成洞察文档，保存到 `Writing/周月刊洞察/`

**Todo使用示例**：

- 输入：脚本返回Todo `[{"content": "应该学会用Figma了", "type": "extracted", "priority": "medium"}]`
- 洞察生成：
  ```
  - [ ] **[工具使用] 开始学习Figma基础**
    - 来源: 副业项目中的"必须得用起来"
    - 推演依据: 已有需求，但非紧急
    - 最小行动: 本周观看1个Figma入门教程
  ```

**推演型Todo示例**：

- 盲点：收集6万字AI评估方法但不实践
- 推演Todo：
  ```
  - [ ] **[AI评估] 给博客添加反馈按钮**
    - 来源：周月刊"想学习AI产品评估方向"
    - 盲点：收集6万字方法论，但自己项目无评测
    - 最小行动：给博客网站添加"是否解决问题"👍按钮（30分钟）
    - 预期成果：收集到至少5个用户反馈
  ```

**Todo筛选示例**：

假设脚本返回了68个显性Todo + 3个推演Todo，需要筛选出5个核心Todo：

1. **AI评估** - 给博客添加反馈按钮（直接解决"收集不实践"的核心盲点）
2. **剪藏实践** - 尝试1个剪藏方法（打破"拥有≠做到"的循环）
3. **关系沟通** - 参与1次深度对话（针对"逃避沟通"的盲点）
4. **工具学习** - 观看1个Figma入门视频30分钟（回应"必须得用起来"的需求）
5. **健康管理** - 用AI分析体检报告（解决"不知所措"的阻碍）

筛选理由：
- 这5个Todo直接对应本周的5个核心盲点
- 每个Todo的最小行动都在30分钟-2小时内可完成
- 覆盖了AI、关系、工具、健康等多个重要领域
- 避免了重复类型的Todo（如只保留1个关系相关的）

## 重要提醒

1. **洞察的深度 > 数量** - 宁可少说几条，也要说到点子上
2. **具体 > 泛泛** - 每条洞察都要有具体内容作为支撑
3. **提问 > 建议** - 让用户自己思考，而非告诉用户怎么做
4. **第三方视角** - 像一个冷静的朋友说出真相，而不是说教
5. **5个Todo > 71个Todo** - 只输出本周最核心的5个Todo，每个Todo的最小行动必须在30分钟-2小时内可完成，减少心理压力，形成正向反馈
